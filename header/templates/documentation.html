Documentation

Welcome to the documentation for the pyStat application. This web application aims to provide a low-code solution for performing various statistical tests. Its core functionality revolves around your uploaded dataset, which is converted into a dataframe upon upload. Numerical columns from your CSV dataset are then displayed on the homepage. You can drag these columns to your chosen widget, created using the buttons on the left side of the homepage. Each widget calculates a specific statistical function on the selected dataset column. The theory and exact functionality of each widget are explained on this page.

Introduction

Statistics is a mathematical field that aims to analyze and make inferences from data. It is primarily concerned with understanding how data is structured and assessing the uncertainty within that data. A crucial aspect of statistics involves drawing samples from a larger population, such as all customers, to make informed conclusions. For these conclusions to be valid, it is essential that the sample is representative of the entire population. This is why a random sample is often preferred, as it helps prevent sample bias. Data can be divided into two main categories, each of which can be further subdivided into several types (Neal & Cavey, 2022):

  1. Numerical Data

    • Discrete: Numerical values that take a finite set of possibilities (e.g., How many siblings do you have?).

    • Continuous: Numerical values that can take any value within a continuous range (e.g., height).

  2. Categorical Data

    • Ordinal: Categorical values that have a natural order (e.g., education level).

    • Nominal: Categorical values that do not have a natural order (e.g., eye color).

    • Binary: Data with only two possible outcomes (e.g., pass/fail, yes/no).

Summary Statistics

Any numerical value calculated from data is called a statistic. Therefore, statistics are functions of data. Before calculating complex statistical functions from your data, it is important to gain some general information about your data. This information is obtained by calculating simple summary statistics, which can be divided into two types: measures of location and measures of spread (Neal & Cavey, 2022).

  • Measures of location aim to calculate a value that is typical for the data, such as the mean, mode, and median.

  • Measures of spread aim to explain how variable your data is. Included in this category are metrics like the variance, standard deviation, range, and interquartile range.

In this web application, you can create a "Summary Statistic Widget." The purpose of this widget is to calculate various summary statistics for a selected column of your uploaded dataset. The widget calculates the following metrics:

  • Mean: The average of all values.

  • Median: The middle value when all values are ordered from smallest to largest.

  • Mode: The most frequently occurring value in the dataset.

  • Variance: A measure of the spread of the values in the dataset, indicating how far each value is from the mean.

  • Standard Deviation: The square root of the variance, providing a measure of the spread of the values around the mean in the same units as the data.

  • Count: The total number of values in the dataset column.

  • Min: The smallest value.

  • Max: The largest value.

These simple summary statistics provide a quick overview and deeper insight into the characteristics of your data.

Hypothesis Testing

Hypothesis testing aims to determine if there is enough evidence in a sample of data to infer that a certain condition is true for the entire population. In other words, it is a form of statistical inference. A standard hypothesis test typically follows these steps:

  • Formulate the null hypothesis (H0), which represents your assumption about the data.

  • Formulate the alternative hypothesis (H1), which is the hypothesis you want to test.

  • Calculate the p-value based on your statistical test.

  • Compare the p-value to your chosen level of significance, often denoted as α.

  • Accept or reject your hypotheses based on the comparison.

The p-value indicates the likelihood of observing the sample values. A low p-value suggests that the null hypothesis can be rejected. For example, in a test for normality, the p-value indicates how likely the results are if the data were normally distributed. A low p-value in this case suggests that the data is not normally distributed. The significance level, typically set at 0.05, is chosen to minimize type 2 errors (false negatives) (Anderson, D. R., Sweeny, D. J., & Williams, n.d.).

Normality Test

The normality test aims to determine if a dataset is normally distributed by using the normal hypothesis testing workflow, which is as follows:

  • H0: The sample comes from a normal distribution.

  • H1: The sample does not come from a normal distribution.

  • Statistical test: \( s^2 + k^2 \) (where \( s \) is the z-score returned by skewtest and \( k \) is the z-score returned by kurtosistest). More information can be found on SciPy.

T Test

The ttest aims to test if the means of two indepdenent samples are significantly different from each other. It follows the normal hypothesis testing workflow, which is as follows:

  • H0: The sample have identical averages

  • H1: The sample do not have identical averages

  • The t-statistic. More information can be found on SciPy.
